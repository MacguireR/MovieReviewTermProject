Indexes to Create

-- Index 1: Speed up review lookups by movie (used on movie detail page)
CREATE INDEX idx_review_movieID ON Review(movieID);

-- Index 2: Speed up movie lookups by genre (used on genre page)
CREATE INDEX idx_moviegenre_genreID ON MovieGenre(genreID);


Affected Queries
----------------
Query 1 (affected by idx_review_movieID):
  SELECT u.username, r.rating, r.comment, r.date
  FROM Review r
  JOIN User u ON r.userID = u.userID
  WHERE r.movieID = ?
  Page: /movies/{id}

Query 2 (affected by idx_moviegenre_genreID):
  SELECT m.movieID, m.title, m.releaseDate
  FROM Movie m
  JOIN MovieGenre mg ON m.movieID = mg.movieID
  WHERE mg.genreID = ?
  Page: /genres/{id}


Timing Results - Insights Page Query (EXPLAIN ANALYZE)
-------------------------------------------------------
Query measured:
  SELECT m.movieID, m.title,
         AVG(r.rating) AS avgRating,
         COUNT(r.reviewID) AS reviewCount
  FROM Movie m
  LEFT JOIN Review r ON m.movieID = r.movieID
  GROUP BY m.movieID, m.title
  ORDER BY avgRating DESC
  LIMIT 100;

  Before indexes: 79.9 ms
  After indexes:  9.05 ms
  Improvement:    ~88% faster

Indexes applied (see indexes.sql):
  CREATE INDEX idx_review_movieid ON Review(movieID);
  CREATE INDEX idx_review_movieid_rating ON Review(movieID, rating);

Optimizer behavior change:
  Before: Left hash join with full table scans on both Movie and Review
  After:  Nested loop left join using covering index lookup on
          idx_review_movieid_rating — index satisfied AVG(rating) and
          COUNT without touching the base table rows
